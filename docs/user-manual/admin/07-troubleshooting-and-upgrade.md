# 故障排查与升级

## 常见问题

| 症状 | 优先检查项 |
|:---|:---|
| 登录后白屏 | 浏览器 Network、`/api/v1/auth/me` 是否成功 |
| 端口冲突 | `deploy/.env` 中的端口配置 |
| 数据库迁移失败 | `docker compose logs migrate` |
| AI 返回空内容 | 模型连通性、路由 max_tokens、上游模型行为 |
| 通知异常 | 系统参数中的通知配置 |

## 升级

```bash
cd deploy
./scripts/upgrade.sh
```

标准升级流程会自动完成：

1. 备份当前数据
2. 拉取或构建新版本
3. 执行数据库迁移
4. 运行健康检查
5. 失败时自动回滚

## 升级建议

- 使用已通过 CI 的 GitHub Release 版本
- 升级前确认备份存在
- 升级后优先抽检登录、审批、采购单、发票和付款相关流程
