# 欢迎使用 Mica（觅采）

**Mica（觅采）** 是一套面向企业内部采购协作的轻量系统，覆盖从提交需求、审批、下单，到收货、开票、付款追踪的完整流程。无论你是业务申请人、采购同事、审批人、财务审核，还是系统管理员，都可以从这里快速找到自己的入口。

---

## 先从哪里开始？

如果你是第一次接触 Mica，建议按下面的顺序阅读：

1. [概述](00-introduction.md) —— 先理解系统解决什么问题、整体长什么样。
2. [角色与权限](01-roles-and-permissions.md) —— 明确你所在角色能看什么、能做什么。
3. [各角色工作流](03-per-role-workflow.md) —— 直接进入你的日常操作路径。

如果你只是想尽快完成手头工作，也可以直接按角色跳转：

| 角色 | 建议入口 | 大致阅读时间 |
|:---|:---|:---|
| 首次使用的普通业务用户 | [概述](00-introduction.md) → [角色与权限](01-roles-and-permissions.md) | 10 分钟 |
| IT 采购员 | [各角色工作流](03-per-role-workflow.md) | 10 分钟 |
| 部门负责人 / 审批人 | [各角色工作流](03-per-role-workflow.md) | 10 分钟 |
| 财务审核 | [各角色工作流](03-per-role-workflow.md) | 10 分钟 |
| 采购经理 | [各角色工作流](03-per-role-workflow.md) | 10 分钟 |
| 系统管理员 | [管理员指南](admin/00-overview.md) | 15-20 分钟 |

## 30 秒快速开始

1. 打开浏览器，访问公司分配的 Mica 地址。
2. 使用管理员或 IT 团队提供的账号登录。
3. 查看右上角的角色信息与主题 / 语言切换入口。
4. 如果不确定下一步做什么，优先打开 [各角色工作流](03-per-role-workflow.md)。

!!! tip "使用体验小提示"
    Mica 支持 **中文 / English** 与 **浅色 / 暗色主题**。你的选择会在浏览器中自动保留。

## 这份文档能帮你什么？

- **快速上手**：帮助新用户在最短时间内理解采购流程和常用页面。
- **按角色操作**：不同角色只看与自己职责有关的内容，不必一次性啃完整手册。
- **按功能查阅**：遇到某个功能不会用时，可以直接去 [功能参考](04-features.md) 检索。
- **排查常见问题**：如果操作结果和预期不一致，先看 [常见问题](05-faq.md)。
- **管理与运维**：系统部署、SSO、HTTPS、备份恢复等内容已整理进新的 [管理员指南](admin/00-overview.md)。

## 常用术语

| 缩写 / 术语 | 含义 |
|:---|:---|
| **PR** | 采购申请（Purchase Requisition），采购流程的起点 |
| **PO** | 采购订单（Purchase Order），正式发给供应商的订单 |
| **SKU** | 标准物料编号，用于统一物料命名和价格跟踪 |
| **RFQ** | 询价流程（Request for Quotation），用于供应商报价收集与比价 |
| **三单匹配** | 采购订单、收货记录、发票三方数据核对 |
| **付款计划** | 按合同或订单拆分的预计付款时间与金额安排 |

## 需要帮助？

- **系统里找不到某个页面或记录？** → 先试试顶部 **全局搜索**（快捷键 `Ctrl+K`）。
- **不知道自己该走哪条流程？** → 打开 [各角色工作流](03-per-role-workflow.md)。
- **想了解某个页面字段是什么意思？** → 查看 [功能参考](04-features.md)。
- **文档里没有覆盖你的问题？** → 联系系统管理员或内部采购负责人。
- **怀疑遇到了 bug？** → 记录页面、操作步骤和报错信息后反馈给管理员。
