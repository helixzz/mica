# 第 0 章｜概述

> Mica（觅采）用户手册 · 第 0 章

## 1. 系统简介

**Mica（觅采）** 是一套面向企业内部的采购管理系统（Internal Procurement Management System），覆盖从采购申请到发票付款的完整业务主线。它以 IT 部门为首要服务对象，并渐进式扩展到其他职能部门。与通用采购 SaaS 不同，Mica 采用深度定制路线，把一家公司真实的工作流沉淀到系统里，让日常采购变得更快、更清楚、更可追溯。如果你是第一次登录 Mica，不需要看完整本手册——先读这一章建立认知，然后按你的角色跳到对应章节即可上手。

## 2. 设计理念

> **效率优先 > 扩展性 > 标准化管控**
> 系统服务于人，而不是限制人。

这一条排序决定了 Mica 的所有产品取舍：

- **效率优先**：能一步做完的操作绝不让用户点第二次；表单默认值、AI 辅助、批量动作都服务于这一点。
- **扩展性**：字段、角色、审批阈值、公司主体都可以随业务长大而增加，但不会为了"未来可能会用到"强行复杂化当下。
- **标准化管控**：合规与审计是底线，但不应反过来绑架效率——所以 Mica 把管控落在**关键节点**（审批、付款、审计日志），而不是落在每一个字段上。

简而言之：Mica 希望你感觉它顺手，而不是感觉它在"管你"。

## 3. 适用场景与用户规模

Mica 是为以下场景设计的，不追求也不适合所有企业：

- **组织规模**：单公司主体 **< 100 名员工**；多公司主体集团可复用同一套部署。
- **业务规模**：月均采购单 **< 300 单**。超过这个量级建议考虑专业 ERP。
- **首要用户**：**IT 部门**（硬件、软件、云资源采购），后续渐进覆盖其他部门。
- **多法人支持**：内置 `companies` 多主体模型，一套系统服务多家关联公司，数据按公司隔离、按用户权限汇总。
- **多币种预留**：币种字段已贯穿 PR / PO / 合同 / 付款 / 发票，当前阶段默认 `CNY`，未来启用多币种无需改表结构。

## 4. 核心能力总览

- 采购申请（PR）→ 审批 → 采购订单（PO）→ 合同 → 交货 → 发票 → 付款 完整主线，一条链路闭环
- 多批次交货 / 多次付款 / 跨 PO 发票 三种分批模型原生支持
- 审批引擎按**金额阈值**自动路由到部门负责人或采购经理，单级审批、路径清晰
- **字段级 + 行级**双重权限：不同角色看到的是同一条记录的不同切面
- 中英双语界面，浏览器语言自动检测，用户可手动切换
- LLM 辅助能力：表单润色、SKU 智能推荐（更多能力开发中）
- **发票上传 + AI 智能提取**：支持 PDF / OFD / XML / 图片，6 级分层提取（XML 原件 99% 置信度，扫描件走 LLM Vision）
- **管理员控制台**：LLM 模型在线配置与测试、AI 调用审计、系统参数、用户权限管理
- 多公司主体（多法人） + 多币种预留，支持小集团场景
- 所有关键操作均写入审计日志（`audit_logs`），可追溯到具体用户和时间

## 5. 快速开始（开发环境）

1. 启动本地开发栈（详见 [`../QUICKSTART.md`](../QUICKSTART.md)）：

   ```bash
   cd deploy
   ./scripts/dev-up.sh
   ```

2. 约 90 秒后，在浏览器打开 `http://<host>:8900`（本机可用 `http://localhost:8900`）。
3. 使用种子账号 `alice / MicaDev2026!` 登录，首次进入会落到"我的工作台"。
4. 如果你是 IT 采购员，点左侧导航的"采购申请" → `新建`，即可走完第一单 PR → 审批 → PO 的完整链路。

> 注意：`MicaDev2026!` 仅用于本地开发种子数据，**不要**用在任何联调/预发/生产环境。生产环境请遵循运维文档的密码与凭据管理规范。

## 6. 术语速查表

| 简写 / 术语 | 中文（English） | 释义 |
| :--- | :--- | :--- |
| `PR` | 采购申请（Purchase Requisition） | 内部发起的采购需求，需经审批后方可转为订单 |
| `PO` | 采购订单（Purchase Order） | 向供应商下达的正式订单，源自已批准的 PR |
| `GR` | 收货（Goods Receipt / Shipment） | 供应商发货后的到货登记，一个 PO 可有多个批次 |
| `三单匹配` | Three-way Matching | 发票与 PO、收货之间的金额/数量校验 |
| `PR 状态` | PR Status | `draft` / `submitted` / `approved` / `rejected` / `returned` / `cancelled` / `converted` |
| `PO 状态` | PO Status | `draft` / `confirmed` / `partially_received` / `fully_received` / `closed` / `cancelled` |
| `PO 进度百分比` | PO Progress % | 基于已收数量 / 已付金额 / 已开票金额计算的履约进度 |
| `审批实例` | Approval Instance | 单次审批流程的运行态对象（`approval_instances`） |
| `公司主体` | Company / Legal Entity | 法人单位，Mica 内所有业务数据归属某一公司主体 |
| `字段级权限` | Field-level Authorization | 同一条记录中，不同角色可见的字段集合不同 |
