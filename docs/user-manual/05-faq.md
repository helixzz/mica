# 第 5 章｜常见问题

本章整理日常使用中高频出现的疑问，按场景分组。找不到答案？请联系管理员。

---

## 登录与权限

**Q：忘记密码怎么办？**
A：当前版本不支持用户自助重置。请联系系统管理员在后台重置，然后首次登录后尽快修改。找回邮箱 / 短信通道属于后续规划。

**Q：为什么我的角色看不到某些菜单？**
A：菜单按角色显示。例如"待办审批"仅审批人可见、财务相关入口仅 `finance_auditor` / `admin` 可见。如果你认为应该有权限却看不到，先确认自己的角色归属，再联系管理员调整。参考 [功能参考 · 权限矩阵](./04-features.md#权限矩阵综合)。

**Q：我切换成英文了，刷新后又变回中文？**
A：语言偏好优先从 `localStorage` 读取。如果浏览器禁用了本地存储（隐身模式、企业安全策略），就只能依赖 `cookie` 或浏览器语言。建议关闭无痕模式、允许站点 Cookie。完整优先级见 [功能参考 · 身份与登录](./04-features.md#身份与登录)。

**Q：我能看到记录，但有些字段是空白/缺失，是 bug 吗？**
A：不是。系统按角色做字段级屏蔽。例如 `dept_manager` 查看发票只能看到 8 个字段，其他字段会被 API 过滤掉。完整清单见 [功能参考 · 权限矩阵](./04-features.md#权限矩阵综合)；若业务上确需查看，请向管理员申请角色调整。

## 采购申请

**Q：为什么 PR 提交后就不能改了？**
A：提交后 PR 状态由 `DRAFT` 变为 `SUBMITTED`，此时已进入审批流，允许随意修改会让审批人看到的版本不一致。**正确做法**：让审批人"退回"，PR 状态会变为 `RETURNED`，此时即可重新编辑。详见 [功能参考 · 采购申请](./04-features.md#采购申请-pr)。

**Q：我的 PR 被退回了，接下来怎么操作？**
A：在"我的采购申请"列表里找到该 PR，状态为 `RETURNED` 的可以直接点击编辑，修改后再次"提交审批"即进入新一轮审批。审批人的退回批注会显示在详情页最上方。

**Q：一个 PR 能同时包含多家供应商的物品吗？**
A：PR 阶段可以（明细行的 `supplier_id` 可以各不相同），**但转 PO 时只支持单供应商**——一个 PO 对应一家供应商。解决方法是按供应商拆分成多个 PR 分别提交审批。参考 [功能参考 · 采购订单](./04-features.md#采购订单-po)。

**Q：我能批量导入 PR 吗（比如 Excel）？**
A：暂不支持。当前版本只能通过 Web 界面逐条录入。批量导入 / Excel 导出在规划中但尚无明确排期。

## 审批

**Q：我是部门负责人，但没收到这个 PR 的待办？**
A：先检查两点：(1) 提交人的 `department_id` 是否和你一致——审批路由优先派给**同部门**的部门负责人；(2) PR 总金额是否 ≥ ¥100,000——此时会先找采购经理。详见 [功能参考 · 审批引擎](./04-features.md#审批引擎)。

**Q：金额很大的 PR 谁来审批？**
A：`≥ ¥100,000` 的 PR 会自动派给**采购经理（`procurement_mgr`）**；若未配置采购经理，会回退到部门负责人，再回退到 `admin`。**当前是单级审批**，并非多级会签。

**Q：审批人出差 / 休假不在，审批卡住了怎么办？**
A：当前版本**尚未提供审批人代理 / 转交机制**。临时方案：请管理员修改目标用户的角色或部门配置，或由 `admin` 直接代审；中长期代理功能在后续版本规划中。

**Q：我能撤回已提交但还没审批的 PR 吗？**
A：可以。在 PR 详情页点击"撤回"，PR 会回到 `DRAFT` 状态，此时可以编辑或删除。撤回后原审批任务会作废。

## 订单与交货

**Q：一个 PO 怎么拆成多批交货？**
A：无需事先"拆分"，直接多次创建交货批次即可。每次录入实际到货数量，系统自动累加 `qty_received` 并更新 PO 状态为 `PARTIALLY_RECEIVED` 或 `FULLY_RECEIVED`。`batch_no` 从 1 起自动递增。详见 [功能参考 · 交货批次](./04-features.md#交货批次-shipment)。

**Q：IT 硬件的序列号必须录入吗？**
A：不强制。`shipment_items.serials[]` 是可选字段，系统不校验数量与序列号个数是否一致。建议对电脑 / 服务器 / 网络设备等需要资产追溯的物品录入；通用消耗品可留空。

**Q：实际到货数量比订单多 / 少，怎么处理？**
A：**少于**订单：正常多批次补齐即可；长期不补齐可将 PO 状态保持在 `PARTIALLY_RECEIVED`，或联系采购经理决定是否以当前数量收官。**多于**订单：当前版本不支持直接超收，请先和供应商对齐差异后，必要时走新一笔 PR/PO。

## 发票与付款

**Q：发票金额比订单金额大，正常吗？**
A：可能正常。发票包含**运费（`freight`）、税费附加（`tax_surcharge`）、调整项（`adjustment`）**等非 product 行，合计自然可能超过订单本体。系统会把超过 PO 行剩余额度的 product 行标为 `warn`（软警告），但**不阻断保存**，供财务复核。详见 [功能参考 · 发票](./04-features.md#发票-invoice)。

**Q：一张发票能跨多个 PO 吗？**
A：**可以**，这是发票模块的核心设计之一。每行通过 `po_id + po_item_id` 关联不同 PO 的行，系统会分别回写各自 PO 的 `amount_invoiced`。参见 [功能参考 · 发票 · 跨 PO 支持](./04-features.md#跨-po-支持重点)。

**Q：付款已确认后能撤回吗？**
A：当前版本**不提供"反确认"按钮**，以避免财务数据反复变动。若确认有误，请联系管理员通过内部流程处理（通常是冲销 + 新建一笔正确记录）。

**Q：开票进度显示超过 100%，这是 bug 吗？**
A：不是 bug，是信号。说明已开票金额超过订单总额，常见原因是运费 / 税费附加 / 调整项计入了同一张发票。系统以此提示财务核对而非拦截。真正的 product 行超额会被额外标为 `warn`，请重点核查那些标记。

**Q：能按客户 / 月份导出发票 Excel 吗？**
A：暂不支持前端一键导出。可通过后端 API 拉取数据自行汇总；内置导出功能在后续版本规划中。

## AI 功能

**Q：点了"AI 润色"，返回内容是空的 / 是占位文本？**
A：当前默认路由到 `demo-mock` 模型（纯演示用），不会产生真实语义的补全。**要接入真实 LLM**，请管理员在 Admin 控制台 → "LLM 模型" 标签页配置模型与 API Key。详见 [功能参考 · AI 辅助功能](./04-features.md#ai-辅助功能)。

**Q：我怎么切换到真实的 LLM（如 GPT-4 / DeepSeek / 通义 / 豆包）？**
A：普通用户无法自行配置，需要管理员在 Admin 控制台 → "LLM 模型" 标签页新增模型 + 填写 API Key，然后在 "AI 场景路由" 标签页把对应 `feature_code` 路由到新模型。控制台已提供可视化编辑 + 测试连通性功能。

**Q：AI 调用会不会产生费用？**
A：`demo-mock` 不产生任何外部费用。一旦接入真实 LLM，每次调用按 token 计费（由模型提供商收取）。系统在 `ai_call_logs` 表记录每次调用的 token 数与耗时，管理员可按此核算成本。

**Q：AI 生成的内容我能直接用吗？**
A：生成内容**仅作辅助建议**，最终由你确认。系统不会自动替你提交表单，需要人工检查后保存。

## 反馈与支持

- **Bug 或改进建议**：通过公司内部 Git 仓库（或 Issue 追踪系统）提交 issue，描述复现步骤、预期行为、实际行为及截图。
- **紧急问题**（如无法登录、审批流程异常、数据不一致）：直接联系系统管理员。
- **功能需求**：同样通过 issue 提交，标记为 `enhancement`；产品会定期评估并纳入版本规划。

建议提交 issue 时附上：使用的角色、操作时间、出错单据编号，便于快速定位。
