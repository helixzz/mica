# 第 5 章｜常见问题

本章整理日常使用中最常见的问题，按场景分组。若这里仍找不到答案，请联系系统管理员。

---

## 登录与权限

**Q：忘记密码怎么办？**  
A：当前版本不支持用户自助重置。请联系系统管理员在后台重置，然后在首次登录后尽快修改。找回邮箱 / 短信通道属于后续规划。

**Q：为什么我的角色看不到某些菜单？**  
A：菜单按角色显示。例如“待办审批”仅审批人可见，财务相关入口通常只对 `finance_auditor` / `admin` 可见。如果你认为自己应该有权限却看不到，先确认角色归属，再联系管理员调整。可配合 [功能参考](./04-features.md) 一起排查。

**Q：我切换成英文了，刷新后又变回中文？**  
A：语言偏好优先从 `localStorage` 读取；如果浏览器禁用了本地存储，就只能依赖 cookie 或浏览器语言。建议关闭无痕模式并允许站点 Cookie。相关机制可参考 [功能参考](./04-features.md)。

**Q：我能看到记录，但有些字段是空白 / 缺失，是 bug 吗？**  
A：通常不是。系统会按角色做字段级屏蔽。比如某些角色查看发票时只能看到部分字段，其余字段会被 API 过滤掉。若业务上确需查看，请联系管理员调整角色或权限策略。

## 采购申请

**Q：为什么 PR 提交后就不能改了？**  
A：提交后 PR 状态从 `DRAFT` 变为 `SUBMITTED`，此时已经进入审批流。若允许随意修改，会让审批人看到的版本不一致。正确做法是让审批人“退回”，PR 进入 `RETURNED` 后再编辑。

**Q：我的 PR 被退回了，接下来怎么操作？**  
A：在“我的采购申请”列表中找到该 PR，状态为 `RETURNED` 的记录可以直接编辑。修改后再次“提交审批”即可进入新一轮审批。

**Q：一个 PR 能同时包含多家供应商的物品吗？**  
A：PR 阶段可以，但转 PO 时当前仍以**单供应商**为前提。一个 PO 对应一家供应商；如果需求来自多家供应商，建议拆成多个 PR 分别提交审批。

**Q：我能批量导入 PR 吗（比如 Excel）？**  
A：当前版本还不支持前端批量导入。现阶段仍以 Web 页面逐条录入为主。

## 审批

**Q：我是部门负责人，但没收到这个 PR 的待办？**  
A：先检查两点：一是提交人的 `department_id` 是否和你一致；二是 PR 总金额是否已经走到采购经理通道。金额较大时，系统会优先路由给 `procurement_mgr`。

**Q：金额很大的 PR 谁来审批？**  
A：通常由 `procurement_mgr` 审批；如果未配置采购经理，系统会按兜底规则回退到其他审批角色。

**Q：审批人出差 / 休假不在，审批卡住了怎么办？**  
A：请联系管理员介入，按当前组织规则临时调整角色、部门或委托关系。是否启用代理审批取决于当前系统配置与版本能力。

**Q：我能撤回已提交但还没审批的 PR 吗？**  
A：可以。在 PR 详情页点击“撤回”后，PR 会回到 `DRAFT`，此时即可继续编辑或删除。

## 订单与交货

**Q：一个 PO 怎么拆成多批交货？**  
A：无需事先拆分，只需多次创建交货批次。系统会根据每次录入的实际到货数量自动累计 `qty_received`，并更新 PO 状态为 `PARTIALLY_RECEIVED` 或 `FULLY_RECEIVED`。

**Q：IT 硬件的序列号必须录入吗？**  
A：不是强制要求，但对电脑、服务器、网络设备等需要资产追踪的物品，建议录入。普通耗材可以留空。

**Q：实际到货数量比订单多 / 少，怎么处理？**  
A：少于订单时，通常按多批次补齐即可；多于订单时，建议先和供应商确认差异，必要时通过新的 PR / PO 处理，不建议直接超收。

## 发票与付款

**Q：发票金额比订单金额大，正常吗？**  
A：可能正常。运费、税费附加、调整项等非 product 行会让发票总额高于订单本体。系统会对异常 product 行给出 `warn` 软警告，供财务复核。

**Q：一张发票能跨多个 PO 吗？**  
A：可以。发票模块支持通过 `po_id + po_item_id` 关联不同 PO 的行，并分别回写对应 PO 的开票进度。

**Q：付款已确认后能撤回吗？**  
A：当前版本不提供“反确认”按钮。若确认有误，请按内部财务流程处理，通常是冲销后重新登记正确记录。

**Q：开票进度超过 100% 是 bug 吗？**  
A：不一定。多数情况下说明发票中包含运费、税费附加或调整项。它更像一个提醒信号，而不是系统错误。

## AI 功能

**Q：点了 “AI 润色”，返回内容是空的 / 是占位文本？**  
A：如果当前路由到演示模型（如 `demo-mock`），就不会生成真实语义补全。要接入真实 LLM，请让管理员在系统管理中配置模型、API Key 与 AI 场景路由。

**Q：我怎么切换到真实的 LLM（如 GPT-4 / DeepSeek / 通义 / 豆包）？**  
A：普通用户无法自行切换。需要管理员在 **系统管理 → LLM 模型 / AI 场景路由** 中完成配置。

**Q：AI 调用会不会产生费用？**  
A：演示模型不会产生外部费用；接入真实模型后，费用通常按模型供应商的 token 计费规则计算。

**Q：AI 生成的内容我能直接用吗？**  
A：建议始终人工复核。AI 输出是辅助建议，不应跳过业务判断直接提交。

## 反馈与支持

- **Bug 或改进建议**：通过内部 Git / Issue 系统提交，尽量附上复现步骤、预期行为、实际行为和截图。
- **紧急问题**：如无法登录、审批流程异常、数据明显不一致，请直接联系系统管理员。
- **功能需求**：同样通过 issue 提交并标记为 `enhancement`，后续会进入版本规划。

建议反馈时附带：使用角色、操作时间、相关单据编号、报错信息，便于快速定位。
